# Help / Info Response

Use this when a user asks "what does this skill do?", "help", "info", or asks how to demo the prototype.

## Short Description

`agent-control-audit` is a Codex skill for agent assurance. It inspects an agent repository, maps missing guardrails, normalizes or proposes golden datasets, generates DeepEval-first tests, invokes the agent through HTTP/MCP/SDK adapters, and renders a repeatable report.

## What It Needs

- Agent repository path or GitHub repository URL.
- Optional user/team golden dataset in JSONL, JSON, CSV, or XLSX. If none exists, Codex can draft a proposed synthetic seed dataset and ask for approval before using it.
- Optional domain or quality profile, such as `financial`, `financial_aml`, `financial_payments`, or `financial_lending`.
- Optional business/domain expectations: intended use, prohibited use, decision rights, approval gates, policies, SOPs, and evidence standards.
- Optional data/MDM expectations: source systems, entity definitions, relationship rules, data quality rules, redaction, lineage, and approved retrieval boundaries.
- Optional model-governance expectations: owner, validator, risk tier, monitoring, change management, retention, rollback, and governance reporting.
- Optional invocation method: HTTP endpoint, MCP stdio server/tool, SDK/framework object, or project-owned command wrapper.
- API keys or provider credentials only for dynamic runs, configured locally through environment variables or a `.env` file.
- Preferred report format: Markdown, Excel, DOCX, or all.
- Output folder for generated artifacts. If omitted, Codex should choose a clear timestamped run folder such as `outputs/<target-slug>/<YYYY-MM-DD-HHMM>/` and initialize the standard stage folders.

## What It Produces

- Static audit JSON.
- Intake artifacts for scope, domain context, business expectations, data/MDM expectations, governance expectations, and quality profile notes.
- Finding evidence JSONL.
- Proposed seed golden dataset when no dataset exists, pending user approval.
- Normalized golden dataset JSONL.
- DeepEval pytest suite.
- Optional eval summary.
- Report in Markdown, XLSX, or DOCX using a fixed template.

These artifacts are generated by Codex through deterministic helper scripts and organized under stage folders such as `02_static_audit/`, `05_dataset/`, `06_evals/`, and `07_reports/`. The user does not need to supply `--in`, `--out`, `--report`, or DeepEval export flags unless they explicitly want implementation details.

## Help Response Rule

For help/info responses, do not show raw shell commands by default. Explain the user journey and ask for the target repo, output folder, optional dataset, domain profile, invocation method, and report format. If no dataset exists, explain that Codex can propose synthetic seed cases but will wait for approval before treating them as golden. If the user explicitly asks for commands, verify the script arguments against the local script help and use only dataset filenames that exist under `evals/datasets/`.

## Secret Handling

- Static audit does not need API keys.
- Dynamic evals use the target repo's normal provider keys, set locally by the user.
- Never place secrets in golden datasets, generated DeepEval suites, reports, or source edits.
- If keys are missing, stop at static/report generation and mark dynamic assurance as pending.

## Suggested User Prompt

```text
Use the agent-control-audit skill on this repo: <path-or-github-url>.
My golden dataset is here: <path>.
Use the financial_aml quality profile.
Generate the report as Markdown and Excel.
```

When no dataset exists:

```text
Use the agent-control-audit skill on this repo: <path-or-github-url>.
I do not have a golden dataset yet.
Draft a proposed seed dataset and wait for my approval before generating DeepEval tests.
```

For dataset-only review:

```text
Use agent-control-audit to review this golden dataset for AML readiness: <path-to-file>.
Tell me if it is good enough for DeepEval and what the user/team needs to fix.
```

Codex should run the deterministic importer internally. Do not ask the user to understand `--in`, `--out`, or `--report`; explain those only if the user asks about implementation details.

For DeepEval generation:

```text
Use agent-control-audit to generate a DeepEval suite from the normalized dataset.
```

For live DeepEval execution:

```text
Use agent-control-audit to run the DeepEval suite against this agent using the available adapter.
```

Codex should run the deterministic exporter internally. If live execution needs dependencies, an invocation path, or API keys, ask for those in plain language.

If the user does not specify a report format, ask:

```text
Which report format do you want: Markdown, Excel, DOCX, or all?
```

Default to Markdown plus Excel when the user says "whatever is best".
